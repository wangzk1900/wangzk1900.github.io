顺序存储结构

顺序存储结构是基于数组的。把根节点存储在下标 i = 1的位置，它左子节点存储在下标 2*i = 2 的位置，右子节点存储在 2*i + 1 = 3 的位置。以此类推，B 节点的左子节点存储在 2*i = 4 的位置，右子节点存储在 2*i + 1 = 5 的位置。
因此，如果数据 X 存储在下标为 i 的位置，那么它的两个子节点分别存储在 2*i 和 2*i + 1 的位置，它的父节点存储在 i/2 的位置。

什么是树？

树是一种非线性结构，节点之间是一对多的关系。如下图所示：

树中的节点分为三类：

  • 根节点——只有子节点，没有父节点。上图中 A 是根节点。
  • 中间节点——既有父节点，又有子节点。例如上图中的 D、E、G、H。
  • 叶（子）节点——只有父节点，没有子节点。例如上图中的 K、J、F。

节点之间的关系：

  • 父节点——若一个节点有子节点，则这个节点是其子节点的父节点。在上图中，A 是 B 和 C 的父节点。
  • 子节点——若一个节点有父节点，则这个节点是其父节点的子节点。在上图中，B 和 C 是 A 的子节点。
  • 兄弟节点——具有相同父节点的节点互称为兄弟节点。在上图中， B 和 C 是互为对方的兄弟节点。

节点的性质：

  • 节点的高度——节点到叶子节点的最长路径（边数）。
  • 节点的深度——根节点到这个节点所经历的的边的个数。
  • 节点的层数——节点深度 + 1。

树的高度 = 根节点的高度

树的分类

根据节点之间有无顺序进行分类：

  • 无序树：树中任意节点的子节点之间没有顺序关系，这种树称为无序树，也称为自由树。
  • 有序树：树中任意节点的子节点之间有顺序关系，这种树称为有序树。

根据节点子树的数量进行分类：

  • m叉树（m>2）
  • 二叉树：每个节点最多含有两个子树的树称为二叉树，这是应用最多的一种树形结构。

二叉树的分类：

  • 完全二叉树：对于一棵二叉树，假设其深度为d（d>1）。除了第d层外，其它各层的节点数目均已达最大值，且第d层所有节点从左向右连续地紧密排列，这样的二叉树被称为完全二叉树。
  • 满二叉树：所有叶节点都在最底层的完全二叉树。
  • 平衡二叉树：当且仅当任何节点的两棵子树的高度差不大于1的二叉树。
  • 二叉查找树：也称二叉搜索树、有序二叉树，树中任何节点的的值大于其左子树所有节点的值，并且小于其右子树所有节点的值。
  • 平衡二叉查找树（AVL树和红黑树）：满足平衡二叉树要求的二叉查找树。
  • 霍夫曼树：带权路径最短的二叉树称为哈夫曼树或最优二叉树。
  • B树：一种对读写操作进行优化的自平衡的二叉查找树，能够保持数据有序，拥有多于两个子树。

04_链表

发表于 2019-05-22 | 更新于 2019-05-25 | 分类于 数据结构和算法

链表是一种链式存储的线性结构。

链表包括：单向链表、双向链表和循环链表。

和数组一样，链表也支持插入、删除和查找操作。

单向链表

单向链表是最简单的链表，它的节点包含两部分：一部分存储数据，另一部分存储指向下一个节点的指针（地址）。

单向链表中第一个和最后一个节点是两个特殊的节点，通常称为头节点和尾节点。
没有任何节点指向头节点，而尾节点不指向任何节点。

插入
插入操作，只需要修改前驱节点的指针，并将新节点指向后继节点，对应的时间复杂度是O(1)。
但是，若在指定的位置插入新的节点，则还需遍历链表找到相应的前驱节点。

删除
删除操作，只需修改前驱节点的指针，对应的时间复杂度是O(1)。
但是不管是删除指定的节点或随机删除第k个节点，都需要遍历链表找到被删节点的前驱节点。

查找
查找指定的节点或随机访问第k个节点，则需要遍历链表，对应的时间复杂度是O(n)。

遍历
由于单向链表节点中只包含一个指针，因此只能按固定的方向（头节点–>尾节点）遍历链表中的节点。

双向链表

双向链表的节点中不仅包含后继节点的指针，还包含前驱节点的指针。因此，相同长度的双向链表要比单向链表多占用1/2的存储空间。

除了头节点的后继节点之外，没有任何节点指向头节点。
除了尾节点的前驱节点之外，尾节点不指向任何节点。

插入
插入操作，只需要修改前驱节点和后继节点的指针，对应的时间复杂度是O(1)。
但是，若在指定的位置插入新的节点，则需遍历链表找到插入位置的前后节点。

删除
删除操作，只需要修改前驱节点和后继节点的指针，对应的时间复杂度是O(1)。
若要删除指定的节点，则很简单，因为根据被删节点可以找到它的前驱节点和后继节点。
但是若要随机删除第k个节点，则需要先遍历链表找到被删节点。

查找
查找指定的节点或随机访问第k个节点，也需要遍历链表。
随机访问第k个节点时，可以先判断这个节点离链表的哪头近，离哪头近就从哪头开始查找。

遍历
支持双向遍历。

循环链表

在单向链表的基础上，将尾节点指向头节点。

和单向链表相比，循环链表的优点是从链尾到链头比较方便。当处理的数据具有环形结构特点，就特别适合使用循环链表。

双向循环链表

在双向链表的基础上，将尾节点指向头节点。

03_数组

发表于 2019-05-21 | 更新于 2019-05-22 | 分类于 数据结构和算法

数组
它用一组连续的内存空间，来存储一组具有相同类型的数据。

随机访问效率高
根据下标随机访问效率高，根据下标和数据类型的长度很容易就能计算出相应元素的存储地址。时间复杂度是O(1)。
但是，若要查找指定的元素，则需要遍历数组，对应的时间复杂的是O(n)。

插入和删除效率低
插入和删除元素比较低效，因为要保持内存数据的连续性。若在k位置插入元素，则需将k位置及其之后的所有元素后移。若要删除k位置上的元素，则该元素之后的所有元素都要向前移动一个位置。

假如数组中的元素是无序的，我们可以改进插入和删除的方法：

  • 在插入新元素时，只需将k位置的元素移动到数组的末尾，然后将新的元素插入就可以了。
  • 对于删除，可以采用滞后删除的方式。即：先标记被删的元素，然后找时机将它们一块删除。

02_算法分析

发表于 2019-05-20 | 更新于 2019-05-22 | 分类于 数据结构和算法

针对同一个问题，通常可以有不同的算法来解决它，这就需要我们评估这些算法的优劣，选出最合适到的那一个。
要评估算法的优劣，基本可以从以下两个方面来考察：

  • 运行的时间，消耗时间越少，效率越高。
  • 占用的空间，占用空间越小越好。

一个程序占用的空间则和以下两点有关：

  • 采用的数据结构；
  • 数据的规模。
    前者跟使用的算法有关（不同的算法对应不同的数据结构），后者取决于程序本身和输入。

事后统计法

最直接的方法就是把代码跑一遍，通过统计、监控就能得到算法的执行时间和占用的内存大小。

这种方法虽然直观，但是比较麻烦，可能还需要自己编写测试程序来监控算法的执行时间和占用内存大小。
另外，测试结果受数据规模的影响也很大。比如，对于小规模的数据，两种算法的效率相差不大，对于大规模的数据，两者的差别可能会很明显。

时间复杂度

一个程序运行的总时间主要和两点有关：

  • 执行每条语句的耗时；
  • 执行每条语句的频率。
    前者取决于计算机和操作系统，后者取决于程序本身和输入。

时间复杂度是什么

一般情况下计算机、操作系统和程序本身变动很小，最大的变动因素是输入。
不同的程序随着数据规模的增大，其运行时间的增长速度也是不同的。
时间复杂度全称是渐进时间复杂度（asymptotic time complexity），表示算法的运行时间与数据规模之间的增长关系，使用大O符号来表示。

常见的时间复杂度：

  • 常数阶 O(1)
  • 对数阶 O(logn)
  • 线性阶 O(n)
  • 线性对数阶 O(nlogn)
  • 平方阶 O(n^2)
  • 立方阶 O(n^3)
  • 指数阶 O(2^n)
  • 阶乘阶 O(n!)

01_概述

发表于 2019-05-19 | 更新于 2019-05-25 | 分类于 数据结构和算法

基本概念

算法
编写一段计算机程序一般都是实现一种已有的方法来解决某个问题。这种方法大多和使用的编程语言无关——它适用于各种计算机以及编程语言。是这种方法而非计算机程序本身描述了解决问题的步骤。在计算机科学领域，我们用算法这个词来描述一种有限、确定、有效的并适合用计算机程序来实现的解决问题的方法。算法是计算机科学的基础，是这个领域研究的核心。（《算法（第4版）》，第1页）

算法（algorithm）是为求解一个问题需要遵循的、被清楚地指定的简单指令的集合。（《数据结构与算法分析 C语言描述（原书第2版）》，第15页）

数据结构
我们关注的大多数算法都需要适当地组织数据，而为了组织数据就产生了数据结构，数据结构也是计算机科学研究的核心对象，它和算法的关系非常密切。在本书中，我们的观点是数据结构是算法的副产品或是结果，因此要理解算法必须学习数据结构。简单的算法也会产生复杂的数据结构，相应地，复杂的算法也许只需要简单的数据结构。（《算法（第4版）》，第2页）

数据结构是对在计算机内存中（有时在磁盘中）的数据的一种安排。数据结构包括数组、链表、栈、二叉树、哈希表等等。算法是对这些结构中的数据进行各种处理，例如，查找一条特殊的数据项或对数据进行排序。（《Java数据结构和算法》，第1页）

抽象数据类型（ADT）
抽象数据类型（ADT）是一种能够对使用者隐藏数据表示的数据类型。用 Java 类来实现抽象数据类型和用一组静态方法实现一个函数库并没有什么不同。抽象数据类型的主要不同之处在于它将数据和函数的实现关联，并将数据的表示方式隐藏起来。在使用抽象数据类型时，我们的注意力集中在 API 描述的操作上而不会去关心数据的表示；在实现抽象数据类型时，我们的注意力集中在数据本身并将实现对该数据的各种操作。（《算法（第4版）》，第38页）

抽象数据类型（abstract data type，ADT）是一些操作的集合。抽象数据类型是数学的抽象；在ADT的定义中根本没有涉及如何实现操作的集合。这可以看做是模块化设计的扩充。对诸如表、集合、图和它们的操作一起可以看作是抽象数据类型，就像整数、实数和布尔量是数据类型一样。整数、实数及布尔量有与它们相关的操作，而抽象数据类型也有与它们自己相关的操作。（《数据结构与算法分析 C语言描述（原书第2版）》，第31页）

什么是ADT？简单说来，他是一种考虑数据结构的方式：着重于它做了什么，而忽略它是怎么做的。（《Java数据结构和算法》，第150页）

我的理解
算法应该是与编程语言无关的，是一种解决某种问题的方法，我们可以用自然语言、数学公式或者编程语言来描述其具体的实现步骤。
数据结构包括逻辑结构和物理结构（存储结构），不同的数据结构支持不同的操作。我们主要研究不同数据操作的效率，以便在编程时选择合适的数据结构。
抽象数据类型（ADT）是一种自定义的数据类型，它应该包括对数据和操作的描述。
ADT应该相当于Java中的接口，可以用不同的数据结构和算法来实现。例如：栈ADT既可以用数组实现也可以用链表实现。

数据结构

数据结构分为：

  • 逻辑结构 – 是指数据对象中数据元素之间的相互关系。
  • 物理结构 – 是指数据的逻辑结构在计算机中的存储形式。

逻辑结构：

  • 集合结构 – 元素之间无特定的关系。
  • 线性结构 – 元素之间是一对一的关系（只有简单的前后关系）。
  • 树形结构 – 元素之间是一对多的关系。
  • 图形结构 – 元素之间是多对多的关系。

物理结构：

  • 顺序存储结构 – 把数据元素存放在地址连续的存储单元里，其数据间的逻辑关系和物理关系是一致的。
  • 链式存储结构 – 把数据元素存放在任意的存储单元里，这组存储单元可以是连续的，也可以是不连续的。

线性结构

基于不同的存储结构，线性结构可以分为：数组和链表。

数组是顺序存储，大部分高级语言都提供了对数组的支持。
链表是链式存储，需要额外的存储空间存放“指针”，节点之间的逻辑关系是通过“指针”来维系的。

常见的链表结构：

  • 单向链表
  • 双向链表
  • 循环链表
