顾名思义，平衡二叉查找树既满足平衡二叉树的要求又满足二叉查找树的要求。

解决的问题：
普通二叉查找树在频繁的插入、删除的情况下，出现时间复杂度退化的问题。

AVL树

在计算机科学中，AVL树是最早被发明的自平衡二叉查找树。在AVL树中，任一节点对应的两棵子树的最大高度差为1，因此它也被称为高度平衡树。查找、插入和删除在平均和最坏情况下的时间复杂度都是 O(logn)。增加和删除元素的操作则可能需要借由一次或多次树旋转，以实现树的重新平衡。AVL树得名于它的发明者G. M. Adelson-Velsky和Evgenii Landis，他们在1962年的论文《An algorithm for the organization of information》中公开了这一数据结构。 –《维基百科》

假如经过一次插入操作后，某个节点失去了平衡，它的左右树的高度之差变成了2，那么有下面四种情形：

  1. 左左：对该节点左子节点的左子树插入了新数据。
  2. 右右：对该节点右子节点的右子树插入了新数据。
  3. 左右：对该节点左子节点的右子树插入了新数据。
  4. 右左：对该节点右子节点的右子树插入了新数据。

伸展树

伸展树（英语：Splay Tree）是一种能够自我平衡的二叉查找树，它能在均摊 O(logn)的时间内完成基于伸展（Splay）操作的插入、查找、修改和删除操作。它是由丹尼尔·斯立特（Daniel Sleator）和罗伯特·塔扬在1985年发明的[1]。–《维基百科》

设计目的：提高频繁访问节点的访问速度。
调整方法：每次查找之后对树进行调整，把被查的节点移到根节点附近，这被称为伸展操作。

伸展操作由一系列的旋转操作组成，每次旋转会使被访问节点离根节点更近。旋转方法跟AVL树是一样的，包括左旋和右旋。
假设被访问节点是 x，每次旋转操作由下面三个因素决定：

  • x 是其父节点 p 的左子节点还是右子节点。
  • p 是否是根节点。
  • p 是其父节点 g 的左子节点还是右子节点。

三种情况：
p 为根节点。

p 不是根节点，并且 p 和 x 都是 g 的左子节点（或右子节点）

p 不是根节点，并且 p 是左子节点而 x 是右子节点（或者 p 是 右子节点而 x 是左子节点）

二叉查找树，也叫二叉搜索树，其特点是：
对于树中的任意一个节点，其左子树中每个节点的值，都要小于这个节点的值；而其右子树所有节点的值都大于这个节点的值。

常用操作

支持快速查找、插入、删除操作。

查找
从根节点开始，如果要查找的数据比根节点的值小，那就在左子树中递归查找；如果要查找的数据比根节点的值大，那就在右子树中递归查找。

插入
和查找过程类似，首先查找出插入位置，然后将新节点插入。

删除
删除操作就比较麻烦了，分三种情况：

  1. 被删节点没有子节点，只需将其父节点中指向该节点的指针置空（root节点需另外处理）。
  2. 被删节点有一个子节点，只需将其父节点指向该节点的指针改为指向其子节点。
  3. 被删节点有两个子节点，首先需要找出其右子树中最小的节点，然后把这个最小节点放到被删节点的位置上。

顺序存储结构

顺序存储结构是基于数组的。把根节点存储在下标 i = 1的位置，它左子节点存储在下标 2*i = 2 的位置，右子节点存储在 2*i + 1 = 3 的位置。以此类推，B 节点的左子节点存储在 2*i = 4 的位置，右子节点存储在 2*i + 1 = 5 的位置。

什么是树？

树是一种非线性结构，节点之间是一对多的关系。如下图所示：

树中的节点分为三类：

  • 根节点——只有子节点，没有父节点。上图中 A 是根节点。
  • 中间节点——既有父节点，又有子节点。例如上图中的 D、E、G、H。
  • 叶（子）节点——只有父节点，没有子节点。例如上图中的 K、J、F。

节点之间的关系：

  • 父节点——若一个节点有子节点，则这个节点是其子节点的父节点。在上图中，A 是 B 和 C 的父节点。
  • 子节点——若一个节点有父节点，则这个节点是其父节点的子节点。在上图中，B 和 C 是 A 的子节点。
  • 兄弟节点——具有相同父节点的节点互称为兄弟节点。在上图中， B 和 C 是互为对方的兄弟节点。

节点的性质：

  • 节点的高度——节点到叶子节点的最长路径（边数）。
  • 节点的深度——根节点到这个节点所经历的的边的个数。
  • 节点的层数——节点深度 + 1。

树的高度 = 根节点的高度

树的分类

根据节点之间有无顺序进行分类：

  • 无序树：树中任意节点的子节点之间没有顺序关系，这种树称为无序树，也称为自由树。
  • 有序树：树中任意节点的子节点之间有顺序关系，这种树称为有序树。

根据节点子树的数量进行分类：

  • m叉树（m>2）
  • 二叉树：每个节点最多含有两个子树的树称为二叉树，这是应用最多的一种树形结构。

二叉树的分类：

  • 完全二叉树：对于一棵二叉树，假设其深度为d（d>1）。除了第d层外，其它各层的节点数目均已达最大值，且第d层所有节点从左向右连续地紧密排列，这样的二叉树被称为完全二叉树。
  • 满二叉树：所有叶节点都在最底层的完全二叉树。
  • 平衡二叉树：当且仅当任何节点的两棵子树的高度差不大于1的二叉树。
  • 二叉查找树：也称二叉搜索树、有序二叉树，树中任何节点的的值大于其左子树所有节点的值，并且小于其右子树所有节点的值。
  • 平衡二叉查找树（AVL树和红黑树）：满足平衡二叉树要求的二叉查找树。
  • 霍夫曼树：带权路径最短的二叉树称为哈夫曼树或最优二叉树。
  • B树：一种对读写操作进行优化的自平衡的二叉查找树，能够保持数据有序，拥有多于两个子树。
