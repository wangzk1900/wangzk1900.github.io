03_数组

发表于 2019-05-21 | 更新于 2019-05-22 | 分类于 数据结构和算法

数组
它用一组连续的内存空间，来存储一组具有相同类型的数据。

随机访问效率高
根据下标随机访问效率高，根据下标和数据类型的长度很容易就能计算出相应元素的存储地址。时间复杂度是O(1)。
但是，若要查找指定的元素，则需要遍历数组，对应的时间复杂的是O(n)。

插入和删除效率低
插入和删除元素比较低效，因为要保持内存数据的连续性。若在k位置插入元素，则需将k位置及其之后的所有元素后移。若要删除k位置上的元素，则该元素之后的所有元素都要向前移动一个位置。

假如数组中的元素是无序的，我们可以改进插入和删除的方法：

  • 在插入新元素时，只需将k位置的元素移动到数组的末尾，然后将新的元素插入就可以了。
  • 对于删除，可以采用滞后删除的方式。即：先标记被删的元素，然后找时机将它们一块删除。
