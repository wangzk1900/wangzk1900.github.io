02_算法分析

发表于 2019-05-20 | 更新于 2019-06-28 | 分类于 数据结构和算法

针对同一个问题，通常可以有不同的算法来解决它，这就需要我们评估这些算法的优劣，选出最合适到的那一个。
要评估算法的优劣，基本可以从以下两个方面来考察：

  • 运行的时间，消耗时间越少，效率越高。
  • 占用的空间，占用空间越小越好。

一个程序占用的空间则和以下两点有关：

  • 采用的数据结构；
  • 数据的规模。
    前者跟使用的算法有关（不同的算法对应不同的数据结构），后者取决于程序本身和输入。

事后统计法

最直接的方法就是把代码跑一遍，通过统计、监控就能得到算法的执行时间和占用的内存大小。

这种方法虽然直观，但是比较麻烦，可能还需要自己编写测试程序来监控算法的执行时间和占用内存大小。
另外，测试结果受数据规模的影响也很大。比如，对于小规模的数据，两种算法的效率相差不大，对于大规模的数据，两者的差别可能会很明显。

时间复杂度

一个程序运行的总时间主要和两点有关：

  • 执行每条语句的耗时；
  • 执行每条语句的频率。
    前者取决于计算机和操作系统，后者取决于程序本身和输入。

时间复杂度是什么

一般情况下计算机、操作系统和程序本身变动很小，最大的变动因素是输入。
不同的程序随着数据规模的增大，其运行时间的增长速度也是不同的。
时间复杂度全称是渐进时间复杂度（asymptotic time complexity），表示算法的运行时间与数据规模之间的增长关系，使用大O符号来表示。

示例1：
假设每条语句执行的时间都一样，为unit_time。

上面代码总的执行时间是：(2n+2)*unit_time。
它的时间复杂度是：O(n)，表示运行时间是线性增长的。

示例2：

这段代码总的执行时间是：T(n) = (2n^2 + 2n + 3)*unit_time。
它的时间复杂度是：O(n^2)，表示运行时间是平方级别的。
当n很大时，公式中的低阶、常量、系数三部分并不影响增长趋势，所以可以忽略。我们只需要记录一个最大量级就可以了。

常见的时间复杂度

常见的时间复杂度：

  • 常数阶 O(1)
  • 对数阶 O(logn)
  • 线性阶 O(n)
  • 线性对数阶 O(nlogn)
  • 平方阶 O(n^2)
  • 立方阶 O(n^3)
  • 指数阶 O(2^n)
  • 阶乘阶 O(n!)

最好的情况是第一个元素就是要找的数据，这样时间复杂度就是O(1)，即：最好情况时间复杂度（best case time complexity）。
最坏的情况是最后一个元素才是要找的数据，这样时间复杂度才是O(n)，即：最坏情况时间复杂度（worst case time complexity）。

上面两种复杂度对应的是两种极端情况，发生概率不大。我们可以分析平均情况下的时间复杂度，即：平均情况时间复杂度（average case time complexity）。

空间复杂度

空间复杂度全称是渐进空间复杂度（asymptotic space complexity），表示算法的存储空间与数据规模之间的增长关系。
常见的空间复杂度有：O(1)，O(n)，O(n^2)。

01_概述

发表于 2019-05-19 | 更新于 2019-05-25 | 分类于 数据结构和算法

基本概念

算法
编写一段计算机程序一般都是实现一种已有的方法来解决某个问题。这种方法大多和使用的编程语言无关——它适用于各种计算机以及编程语言。是这种方法而非计算机程序本身描述了解决问题的步骤。在计算机科学领域，我们用算法这个词来描述一种有限、确定、有效的并适合用计算机程序来实现的解决问题的方法。算法是计算机科学的基础，是这个领域研究的核心。（《算法（第4版）》，第1页）

算法（algorithm）是为求解一个问题需要遵循的、被清楚地指定的简单指令的集合。（《数据结构与算法分析 C语言描述（原书第2版）》，第15页）

数据结构
我们关注的大多数算法都需要适当地组织数据，而为了组织数据就产生了数据结构，数据结构也是计算机科学研究的核心对象，它和算法的关系非常密切。在本书中，我们的观点是数据结构是算法的副产品或是结果，因此要理解算法必须学习数据结构。简单的算法也会产生复杂的数据结构，相应地，复杂的算法也许只需要简单的数据结构。（《算法（第4版）》，第2页）

数据结构是对在计算机内存中（有时在磁盘中）的数据的一种安排。数据结构包括数组、链表、栈、二叉树、哈希表等等。算法是对这些结构中的数据进行各种处理，例如，查找一条特殊的数据项或对数据进行排序。（《Java数据结构和算法》，第1页）

抽象数据类型（ADT）
抽象数据类型（ADT）是一种能够对使用者隐藏数据表示的数据类型。用 Java 类来实现抽象数据类型和用一组静态方法实现一个函数库并没有什么不同。抽象数据类型的主要不同之处在于它将数据和函数的实现关联，并将数据的表示方式隐藏起来。在使用抽象数据类型时，我们的注意力集中在 API 描述的操作上而不会去关心数据的表示；在实现抽象数据类型时，我们的注意力集中在数据本身并将实现对该数据的各种操作。（《算法（第4版）》，第38页）

抽象数据类型（abstract data type，ADT）是一些操作的集合。抽象数据类型是数学的抽象；在ADT的定义中根本没有涉及如何实现操作的集合。这可以看做是模块化设计的扩充。对诸如表、集合、图和它们的操作一起可以看作是抽象数据类型，就像整数、实数和布尔量是数据类型一样。整数、实数及布尔量有与它们相关的操作，而抽象数据类型也有与它们自己相关的操作。（《数据结构与算法分析 C语言描述（原书第2版）》，第31页）

什么是ADT？简单说来，他是一种考虑数据结构的方式：着重于它做了什么，而忽略它是怎么做的。（《Java数据结构和算法》，第150页）

我的理解
算法应该是与编程语言无关的，是一种解决某种问题的方法，我们可以用自然语言、数学公式或者编程语言来描述其具体的实现步骤。
数据结构包括逻辑结构和物理结构（存储结构），不同的数据结构支持不同的操作。我们主要研究不同数据操作的效率，以便在编程时选择合适的数据结构。
抽象数据类型（ADT）是一种自定义的数据类型，它应该包括对数据和操作的描述。
ADT应该相当于Java中的接口，可以用不同的数据结构和算法来实现。例如：栈ADT既可以用数组实现也可以用链表实现。

数据结构

数据结构分为：

  • 逻辑结构 – 是指数据对象中数据元素之间的相互关系。
  • 物理结构 – 是指数据的逻辑结构在计算机中的存储形式。

逻辑结构：

  • 集合结构 – 元素之间无特定的关系。
  • 线性结构 – 元素之间是一对一的关系（只有简单的前后关系）。
  • 树形结构 – 元素之间是一对多的关系。
  • 图形结构 – 元素之间是多对多的关系。

物理结构：

  • 顺序存储结构 – 把数据元素存放在地址连续的存储单元里，其数据间的逻辑关系和物理关系是一致的。
  • 链式存储结构 – 把数据元素存放在任意的存储单元里，这组存储单元可以是连续的，也可以是不连续的。

线性结构

基于不同的存储结构，线性结构可以分为：数组和链表。

数组是顺序存储，大部分高级语言都提供了对数组的支持。
链表是链式存储，需要额外的存储空间存放“指针”，节点之间的逻辑关系是通过“指针”来维系的。

常见的链表结构：
